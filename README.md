# Design Decisions

## Hybrid Safety Detection

Safety checking combines two approaches:

### Rule-Based Validation

Fast deterministic checks for obvious violations such as:

* illness prediction
* payment pressure
* explicit dangerous claims

### LLM-Based Validation

A second LLM review identifies contextual violations that are difficult to detect using simple keyword matching, especially across English, Hindi, and Hinglish conversations.

This combination provides both speed and flexibility. 
The rule-based checker intentionally detects only deterministic English patterns with near-zero false positives. Multilingual and contextual safety violations are handled by the LLM reviewer, which is better suited for semantic understanding across English, Hindi, and Hinglish.


---

## Chat Count

The combined count of words from both user and assistant is conisdered.

---

## Duplicate Detection

Near-duplicate detection is implemented using **RapidFuzz** with token-based similarity.

RapidFuzz was chosen because it provides a lightweight and effective solution for the small dataset used in this assignment without introducing unnecessary complexity. For a production-scale system with thousands of conversations, embedding-based semantic similarity (e.g., Sentence Transformers + FAISS) would be a more scalable and robust approach for detecting semantic duplicates.

---

## Response Evaluation

Responses are evaluated on:

* Safety
* Helpfulness
* Compassion
* Responsible Astrology
* Overall Quality

The evaluator produces structured scores that can be compared across responses.

# ADDITIONAL WORK:

## Validation & Testing

The `tests/` directory contains additional datasets and utilities used to validate the implementation.

### `checker_tests/`

- **`safe_chats.jsonl`** тАУ Verifies that valid Vedaz conversations pass all validation checks.
- **`unsafe_chats.jsonl`** тАУ Contains intentionally unsafe conversations (death prediction, guaranteed outcomes, payment pressure, etc.) to validate both rule-based and LLM-based safety detection.
- **`duplicate_chats.jsonl`** тАУ Contains exact and near-duplicate conversations to validate duplicate detection.
- **`outputs/`** тАУ Stores the generated checker reports (`.json`) and corresponding console output screenshots (`.png`) for each test.

### `evaluator_tests/`

- **`evaluation_test.json`** тАУ Contains the same questions with both safe and intentionally unsafe responses to verify that the evaluator assigns lower scores to unsafe responses.
- **`validate_evaluator.py`** тАУ Runs the evaluator on the validation dataset.
- **`outputs/`** тАУ Stores the generated evaluation results (`evaluation_results.csv`).

# Limitations

* Rule-based safety detection may miss contextual or multilingual violations.
* LLM-based evaluation may produce slight scoring variations across runs.
* Duplicate detection is lexical and may miss semantic similarities.

---

# Future Improvements

* Expand rule-based safety checks with broader multilingual patterns.
* Fine-tune a lightweight safety classifier for more consistent evaluation.
* Adopt embedding-based semantic duplicate detection (e.g., Sentence Transformers + FAISS).
